PROJECT SPECIFICATION
Build a pixel-perfect Netflix clone with complete functionality including video playback, user authentication, watchlist management, and progress tracking. The application must be fully functional on first deployment with zero additional configuration needed beyond API key integration.​

TECHNICAL STACK
Frontend Framework:

Vite + React 18 + TypeScript

React Router DOM v6 for routing

Context API + useReducer for state management

Axios for HTTP requests

React Icons for UI icons

Styling:

CSS Modules or Styled Components

CSS animations for hover effects and transitions

Mobile-first responsive design (320px to 4K)

External Services:

TMDB API: Content metadata (movies, TV shows, cast, genres, trending)

Vidking API: Video playback via iframe embedding

Firebase: Authentication (email/password, Google OAuth)

Supabase: User data storage (watchlist, viewing progress, profiles)

Core Service Implementations
TMDB Service (services/tmdb.ts):

getTrending() - Fetch trending content

getPopular(type: 'movie' | 'tv') - Popular content by type

getGenres() - All available genres

getByGenre(genreId) - Content filtered by genre

getDetails(id, type) - Full content details

getCredits(id, type) - Cast and crew

getSimilar(id, type) - Similar recommendations

search(query) - Search across all content

Implement request caching with 5-minute expiration​

Vidking Service (services/vidking.ts):

generateEmbedUrl(tmdbId, type, options) - Create iframe URL

parseProgressEvent(event) - Parse postMessage data

Handle episode navigation for TV series

Apply custom color themes dynamically

Firebase Service (services/firebase.ts):

Initialize Firebase app

Email/password authentication methods

Google OAuth provider setup

Session persistence

User state observer

Supabase Service (services/supabase.ts):

Initialize Supabase client

Database schemas:

profiles table: userId, profileId, name, avatar, ageRating

watchlist table: userId, profileId, contentId, contentType, addedAt

viewing_progress table: userId, profileId, contentId, contentType, progress, timestamp, duration, season, episode

CRUD operations for all tables

Real-time subscriptions for live updates
